Posted on 2020-07-15 Edited on 2020-08-01 In OS

I. OS Introduction

1. 운영체제란?

  1. Windows 10, 7, Linux, MS-DOS, Mac OX, iOS
  2. 운영체제 : Operating System
  3. 컴퓨터에 운영체제가 없다면?
    • 컴퓨터: 프로세서(CPU) & 메모리
    • 전원을 켜면 휘발성 메모리(RAM)가 먼저 활성화됨 -> 야생마와 같은 상태
  4. 운영체제의 역할
    • 단일 프로그램 실행
    • 프로그램을 여러개 실행
    • 인쇄, 출력, 저장
  5. 운영체제의 의의
    • 하드웨어 관리
    • 성능 향상 (Performance)
    • 편의성 (Convinence)
    • Control program for Computer

2. Booting

  1. 컴퓨터 구조
    • 프로세서(CPU), 메모리(RAM, ROM), 디스크
    • POST(Power-On Self-Test)
    • 부트 로더(Boot Loader)
  2. 부팅 과정
    • 컴퓨터 Power On > POST(Power-On Self-Test) >Rom (Bootstrap Load) > HDD 안의 OS > 메인메모리(RAM)로 가져옴
  3. 운영체제 구성
    • 관리(Management) 프로그램
    • 프로세서, 메모리, 디스크, 입출력장치 드라이브
    • 커널(Kernal) vs 명령 해석기(Shell, Command Interpreter)
  4. 운영체제의 위상
    • 하드웨어 > 운영체제(커널 > 쉘) > 애플리케이션
  5. OS vs Government
    • 자원 관리자(resource manager)
    • 자원 할당자(resource allocator)
    • 주어진 자원의 활용
    • 직접 일하지 않음
    • 업무별로 부서가 할당됨

3. OS History : 하드웨어 발전 / 운영체제 발전

  1. 시초 : 1940년대 말
    • card reader > memory > processing > line printer
  2. Batch Processing System(일괄처리) : Resident Monitor
  3. Multiprogramming System(다중프로그래밍)
    • 아직까지 컴퓨터는 비싼 자원
    • 빠른 COU vs 느린 i/o -> 메모리에 여러개의 job over loading
    • CPU 스케쥴링, 메모리 관리, 보호가 중요
  4. Tims-Sharing System(시공유 시스템)
    • ms 단위로 강제 전환, interactive system(대화형)
    • 가상 메모리, 프로세스간 통신, 동기화

4. 고등 운영체제

  1. 다중 프로세서(Multiprocessor System)
    • 병렬(parallel)
    • 강결합(tightly-coupled)
    • performance, cost, reliability의 장점이 있음
    • 1 memory, multi CPU
  2. 분산 시스템(Distributed System)
    • 다중 컴퓨터 시스템(multi-computer)
    • 소결합(loosely-coupled)
    • 분산 운영체제(Distributed OS)
    • multi cpu, multi memory, connected by lan
  3. 실시간 시스템(Real-time System)
    • 시간 제약(deadline)이 있는 프로그래밍
    • 공장 자동화(FA), 군사, 공항, 우주, 네비게이션, etc
    • 실시간 운영체제(RTOS)
