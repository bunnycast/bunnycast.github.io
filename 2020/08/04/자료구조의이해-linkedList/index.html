1. Linked List 구조

  • a.k.a 연결 리스트

  • 배열 : 순차적으로 연결된 공간에 데이터를 나열

  • 링크드 리스트는 떨어진 곳에 존재하는 데이터를 화살표로 연결해서 관리하는 구조

  • C 언어에서는 주요한 데이터 구조이지만, 파이썬은 리스트 타입이 링크드 리스트의 기능을 모두 지원

  • 링크드 리스트의 기본 구조와 용어

    • Node : 데이터 저장 단위(데이터, 포인터)
    • pointer : 다음이나 이전 노드와의 연결 정보

      마지막 데이터의 포인터는 null값일 수 있음

  • 출처: wikipedia, https://en.wikipedia.org/wiki/Linked_list

3. 링크드 리스트의 장단점(전통적인 C언어에서의 배열과 링크드 리스트)

  • 장점
    • 미리 데이터 공간 할당할 필요 없음
  • 단점
    • pointer를 위한 별도의 데이터 공간이 추가로 필요해, 저장 공간 효율이 높지 않음
    • 연결 정보를 찾는 시간이 소요돼 접근 속도가 느림
    • 중간에 데이터를 추가/삭제할 시, 앞뒤 데이터의 연결을 재구성해야 함
